3.1

Introduction to CSS & Types of CSS

What CSS is, why we use it, and the three ways to add it to a webpage

What is CSS?

Definition

CSS (Cascading Style Sheets) is a stylesheet language used to control the visual appearance of HTML elements on a web page. While HTML defines the structure and content (what is there), CSS defines the presentation (how it looks) — colours, fonts, sizes, spacing, layouts, animations and more. CSS works by selecting HTML elements and applying style rules to them.

HTML does CSS does
Defines structure (headings, paragraphs, links) Defines appearance (colour, font, size, layout)
<h1>Hello</h1> — puts text on page Makes that h1 red, 32px, bold, centered
Content skeleton The skin, makeup, and outfit
📌 The word "Cascading" means styles flow down from parent to child elements, and when multiple rules target the same element, a specificity priority order decides which one wins.

CSS Syntax

Syntax

A CSS rule has two parts: a selector (which HTML element to target) and a declaration block (what to change). Inside the block, each declaration is a property-value pair separated by a colon, ending with a semicolon. 

/* Basic CSS syntax */
selector {
  property: value;
  property: value;
}

/* Real example */
h1 {
  color: red;
  font-size: 32px;
  text-align: center;
}

Three Types of CSS

Overview

There are three ways to add CSS to a web page: Inline CSS (directly on an element), Internal CSS (in a <style> tag in the <head>), and External CSS (in a separate .css file linked to the HTML). Each has different scope, priority, and best-use situation.

1. Inline CSS

Definition

Inline CSS is written directly inside an HTML element using the style attribute. It applies styles to only that specific element. It has the highest priority — it overrides both internal and external CSS. However, it mixes content with styling and is very hard to maintain at scale. 

<!-- Inline: style="" directly on the element -->
<h1 style="color: blue; font-size: 32px; text-align: center;">Hello World</h1>

<p style="color: gray; font-family: Arial; margin: 10px;">
  Paragraph with inline styles.
</p>

2. Internal CSS

Definition

Internal CSS is written inside a <style> tag placed in the <head> section of the HTML file. It applies styles to elements throughout that single HTML page. Useful for single-page projects or page-specific styles, but cannot be shared across multiple pages. 

<!DOCTYPE html>
<html>
<head>
  <!-- Internal CSS: inside <style> tag in <head> -->
  <style>
    body {
      background-color: #f0f0f0;
      font-family: Arial, sans-serif;
    }
    h1 { color: navy; text-align: center; }
    p  { color: #333; line-height: 1.6; }
  </style>
</head>
<body>
  <h1>Styled with Internal CSS</h1>
</body>
</html>

3. External CSS

Definition

External CSS is written in a separate .css file and linked to the HTML using a <link> tag in the <head>. This is the best practice and most widely used method — the same CSS file can be linked to many HTML pages, making styling consistent and easy to update. It completely separates structure (HTML) from presentation (CSS). 

<!-- HTML file: link the external .css file -->
<head>
  <link rel="stylesheet" href="style.css">
</head>

/* Inside style.css (completely separate file) */
body {
  margin: 0;
  font-family: 'Segoe UI', sans-serif;
  background-color: #fff;
}
h1 { color: purple; }
p  { color: #555; }

Comparison — All Three Types

Type Where Written Scope Priority Best For
Inline Inside element's style="" That one element only Highest ① Quick one-off overrides
Internal <style> in <head> That single HTML page Medium ② Single-page projects
External Separate .css file Any page that links it Lowest ③ Real projects — best practice
⚠️ Priority when rules conflict: Inline > Internal > External. Always use External CSS for real projects. Avoid inline styles — they are impossible to maintain at scale.
